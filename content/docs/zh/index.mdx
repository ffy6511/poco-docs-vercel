Poco 是一个多服务协同的 AI Agent 执行平台，面向需要把编码任务从“发起需求”推进到“沙箱执行、过程观测、产物沉淀”的开发者和团队。它把 Web 交互、后端事实源、任务调度、Agent 执行和回调持久化拆成清晰的服务边界，让长时间、多工具、多端协作的 Agent 工作流更容易运行和追踪。

它的核心特点包括：

- **多服务架构**：由 Frontend、Backend、Executor Manager 和 Executor 协同工作，分别负责交互、状态持久化、任务调度和沙箱执行。
- **Server / Channel 频道协作**：以 server 作为长期协作空间，以 channel、DM 和 thread 承载讨论，让消息、任务、Agent run 和共享产物都能回到同一条协作上下文中。
- **任务生命周期完整**：支持创建任务、调度执行、回写进度、记录工具调用、查看结果和复盘执行过程。
- **沙箱化 Agent 执行**：Agent 在隔离环境中运行命令、读写文件、调用工具和生成产物，降低对主服务和宿主环境的影响。
- **协作与观测优先**：围绕 server、channel、message、task 和 run 组织上下文，让人类与 Agent 的协作过程可追踪。
- **可扩展能力面**：通过 Preset、MCP、Skills、浏览器、GitHub 仓库、本地目录挂载和智能记忆扩展不同类型的工作流。
