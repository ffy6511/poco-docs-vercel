title: 文件上传
description: 让 Agent 在任务中使用真实材料，而不局限于纯文本提示词。
---

Poco 支持文件型输入，让 Agent 可以直接使用截图、报告、文档、数据文件等真实材料。文件上传的意义不只是把附件放进对话里，而是让任务能够建立在材料本身之上，而不是依赖手工转述。

## 上传文件在产品中的角色

上传文件本质上属于任务输入。它们通常服务于当前会话、当前任务，或者当前项目下的一次具体工作。

```mermaid
flowchart LR
    A["Uploaded files"] --> B["Task context"]
    B --> C["Agent work"]
    C --> D["Generated result"]
    D --> E["Shared output if worth reusing"]
```

Poco 在这里区分得很清楚：先用上传文件帮助 agent 理解任务，再判断生成结果中是否存在值得保留、可供后续复用的共享成果。

## 典型场景

- 上传参考文件作为当前任务输入
- 让 Agent 结合指令与文档内容进行分析
- 在一次工作流中处理多种文件格式
- 根据截图、报告或设计文档生成后续成果

## 上传文件、项目文件与共享文件

这三类文件承担的角色并不相同。

| 类型 | 主要作用 | 生命周期 |
| --- | --- | --- |
| 会话上传文件 | 一次任务的输入材料 | 当前会话或当前任务 |
| 项目文件 | 长期复用的固定参考上下文 | 项目周期内持续存在 |
| 共享文件 | 已经沉淀为公共成果、供频道继续协作 | 频道内长期复用 |

这个区分会直接影响使用体验。不是所有上传文件都应该自动变成公共成果；反过来，也不是所有公共成果都还应被当作一次性附件看待。很多时候，文件先作为任务输入存在，只有在真正具备复用价值后，才会进入共享层。
